# Value-Added Service Image Blocks

This directory is generated by `scripts/build_value_added_image_blocks.py`.

## Scope

- Includes embedded images from single-service `.xlsx` files under:
  - `sop/增值服务大全（未上翻）/`
- Skips by default:
  - `sop/增值服务大全（未上翻）/增值业务大全20260105 (2).xlsx`

The skipped workbook is treated as a duplicate summary workbook. The debt-consulting folder's `.xlsx` files currently do not contain embedded images.

## Files

- `images/`: de-duplicated extracted image assets.
- `contact_sheets/`: generated review sheets with `image_id`, service, source, sheet, and anchor labels.
- `image_inventory.json`: source provenance, anchors, duplicate references, and nearby cell text.
- `manual_image_annotations.json`: human/LLM visual-understanding annotations, if present.
- `image_blocks.jsonl`: retrievable supplemental image blocks. Before annotation, blocks are marked `annotation_status=unreviewed`; after annotation they contain `visual_summary`, `key_information`, `importance`, and `prompt_usage`.
- `manifest.json`: generation summary and diagnostics.

## Annotation Dimensions

Each image should be reviewed along these dimensions:

- `image_type`: `app_ui_path`, `backend_console`, `policy_rule`, `agreement_long_text`, `marketing_visual`, `qr_contact`, `reward_table`, `payment_status`, or `other`.
- `importance`: `critical`, `supplemental`, or `low`.
- `visual_summary`: short description of what the image shows.
- `key_information`: concise business facts extracted by visual understanding.
- `prompt_usage`: when this image supplement should be included in the prompt.
- `intent_tags`: coarse tags such as `inquiry`, `cancel`, `refund`, `bill_repayment`, `operation_path`, `escalation`, `contact`.

This stage intentionally does not OCR images. It uses manual or multimodal visual interpretation as a supplement to the text blocks in `../value_added_text/`.
