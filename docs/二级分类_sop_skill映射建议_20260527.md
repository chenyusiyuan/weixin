# 二级分类与 SOP/Skill 映射建议（2026-05-27）

## 口径

本轮按“二级分类”处理：

- 分类标签来源：`标注维度.xlsx` 的二级小结，共 43 类。
- 当前 `golden_test.jsonl` 只覆盖其中 27 类；未覆盖的 16 类仍属于二级分类 label space。
- `skill_id` 不是最终分类标签，只是二级小结背后的可接受 SOP/skill 承接能力。
- 一个二级标签可以接受多个 skill；同一个 skill 也可能被多个二级标签接受。因此不能只靠 `skill -> 二级标签` 的单向表做严谨评测。

状态更新：当前 `scripts/references/merged_intent_skill_mapping.json` 已按这份建议扩展为 43 条 intent 映射；Demo 结果分析按二级名称分组时去重后展示 42 个分组，因为不同一级类下存在同名二级小结。

## 对当前正式 mapping 的修改建议

当前 `scripts/references/merged_intent_skill_mapping.json` 已作为“标注维度二级分类 -> 可接受 skill”评测和 Demo 分组表使用。已按本建议落地的关键改动包括：

1. `催收相关/投诉催收` 增加 `close_pre_reminder`。
2. `催收相关/要求停催` 增加 `close_pre_reminder`。
3. `还款相关/存对公还款` 增加 `repayment_result_query`。
4. `申请咨询/预约借款` 从 `loan_consultation` 调整为 `value_added_service_inquiry`, `loan_termination`, `disbursement_progress`。
5. `信息维护/资料信息修改` 不再映射 `card_rebinding`；新增资料维护 skill 前，应标为能力缺口。

## 二级分类评测注意点

不要用单一 `skill -> 二级标签` 表替代 `二级标签 -> 可接受 skill` 表。原因：

- `clearance_certificate` 既可能服务 `业务办理/结清证明`，也可能是 `还款相关/账单信息查询` 中“已结清查询/证明”的合理承接。
- `loan_termination` 既可能服务 `申请咨询/贷款解约`，也可能服务 `申请咨询/预约借款/取消预约借款`。
- `value_added_service_inquiry` 可能服务 `营销活动/新活动咨询`、`营销活动/会员咨询`，也可能作为会员/增值服务退费前置咨询。

因此二级分类最稳的实现方式是：

1. 分类器直接输出二级小结 label。
2. 若现阶段仍以 skill router 输出为主，则评测时用 `二级标签 -> 多个可接受 skill`，并保留一对多/多对一关系。
3. 对当前 golden 未覆盖的二级分类，报告为 `0 support`，不要纳入准确率分母；但保留在 label space 和覆盖状态表里。
