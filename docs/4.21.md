## 三、语义纠缠

### 3.1 催收簇：collection_complaint ↔ stop_collection

**混淆次数**：双向共 47 次（38+9）

**语义边界**：

- `collection_complaint`：客户对**已发生**的催收行为不满/投诉（情绪化、指责语气）
- `stop_collection`：客户明确要求**停止未来**的催收电话（不涉及投诉品质）

**典型混淆案例**：

| 客户输入                                                 | Gold 标注            | 模型预测               | 分析                                           |
| -------------------------------------------------------- | -------------------- | ---------------------- | ---------------------------------------------- |
| 我的账单已经处理完了，为什么还在打电话？                 | collection_complaint | stop_collection        | 客户在抱怨已发生的催收，但模型理解为要停止电话 |
| 我打这么多电话？别给我打了                               | collection_complaint | stop_collection        | 投诉催收方式，但缺少明确情绪词                 |
| 为什么平台在催我还款，但我已经没有借款了？               | collection_complaint | post_loan_verification | 质疑+投诉边界模糊                              |
| 请停止催收，包括发送短信和威胁                           | stop_collection      | collection_complaint   | 明确要求停止，但出现"威胁"情绪词导致误判       |
| 我想请问，我上次还了一笔款，前几天催收是否可以暂停几天？ | stop_collection      | -                      | 请求暂停催收，表达委婉                         |

**纠缠原因**：

1. 同一客户可能同时具有"投诉"和"要求停止"两种诉求
2. 单句输入缺少上下文，难以区分是"抱怨过去"还是"要求未来"
3. 情绪词（威胁、骚扰）与行为请求（停止、暂停）同时出现造成混淆

---

### 3.2 扣款簇：bill_deduction_query ↔ deduction_issues ↔ repayment_status_issue

**混淆次数**：共 98 次（33+33+32）

**语义边界**：

- `bill_deduction_query`：客户查本期应扣/已扣的账单状态（中性查询）
- `deduction_issues`：客户质疑异常/陌生/多笔扣款（质疑语气）
- `repayment_status_issue`：客户自己发起的还款出问题（断言失败）

**典型混淆案例**：

| 客户输入                                             | Gold 标注              | 模型预测         | 分析                                 |
| ---------------------------------------------------- | ---------------------- | ---------------- | ------------------------------------ |
| 今天早上众安保险怎么又从我这边扣款了？扣了274元。    | bill_deduction_query   | deduction_issues | 查询性质，但"怎么又"带有质疑语气     |
| 你们后台扣了我八十多块钱                             | bill_deduction_query   | deduction_issues | 陈述扣款事实，语义中性但被判定为异常 |
| 我刚刚打电话联系过，是关于代偿的问题                 | post_loan_verification | -                | 单句信息不足                         |
| 为什么我的账户里有一个叫东富的条目？                 | fee_detail_query       | none             | 陌生条目查询，缺少费用关键词         |
| 我已经提前还款了，但系统一直显示还款中，还在继续扣款 | repayment_status_issue | -                | 还款问题+继续扣款，双重语义          |

**纠缠原因**：

1. 扣款相关场景语义高度重叠：查账单 → 质疑异常 → 还款失败 是连续流程
2. 客户表达时常混合多种意图："扣了274元"既是查询也是质疑
3. 单句难以判断客户是"确认金额"还是"质疑异常"

### 3.3 征信簇：credit_inquiry ↔ credit_modification ↔ bill_date_credit_impact

**混淆次数**：共 16 次（7+6+1+2）

**语义边界**：

- `credit_inquiry`：查询征信记录/状态（如"征信显示我有欠款"）
- `credit_modification`：要求修改/消除征信记录（如"想解除关注状态"）
- `bill_date_credit_impact`：询问还款日期对征信的影响（如"逾期一天是否影响信用"）

**典型混淆案例**：

| 客户输入                                                     | Gold 标注               | 模型预测             | 分析                         |
| ------------------------------------------------------------ | ----------------------- | -------------------- | ---------------------------- |
| 我的钱包逾期了，我想一次性还清，但我的征信报告显示是关注状态，我想解除关注状态。 | credit_modification     | early_loan_clearance | 多意图：提前还款+征信修改    |
| 征信显示我没有欠款，债权金额是1005元。我的账户已经结清并关闭了，但征信上仍有一笔1005元的欠款记录 | credit_inquiry          | -                    | 查询征信记录，但隐含修改诉求 |
| 如何处理因还款日期问题导致的逾期记录？                       | credit_inquiry          | -                    | 咨询性质，但隐含修改意图     |
| 我想咨询豆豆钱逾期一天是否会影响信用记录                     | bill_date_credit_impact | -                    | 明确询问影响，此类准确率较高 |

**纠缠原因**：

1. 征信场景往往同时包含查询和修改诉求
2. 客户表达时先说现状（征信有问题）再说需求（想修改），首句可能只包含现状
3. 缺少"我想修改/消除"等明确动词时难以判断

## 四、问题根因总结

### 4.1 核心问题

| 问题类型            | 占比 | 典型表现                       |
| ------------------- | ---- | ------------------------------ |
| **语义边界模糊**    | ~40% | 同一表达可合理映射到多个 skill |
| **单句信息不足**    | ~25% | 仅凭单句无法判断真实意图       |
| **多意图混合**      | ~20% | 客户同时表达 2+ 种诉求         |
| **Gold 标注可争议** | ~10% | 标注本身存在歧义               |
| **模型能力上限**    | ~5%  | LLM 对细粒度判别能力不足       |

### 4.2 具体分析

**1. 语义边界模糊（~40%）**

特征：多个 skill 定义存在语义重叠区域

- 催收簇：投诉 vs 停止 vs 核实身份
- 扣款簇：查账单 vs 质疑异常 vs 还款失败
- 征信簇：查询 vs 修改 vs 影响咨询

影响：模型难以在边界案例上做出正确选择

**2. 单句信息不足（~25%）**

特征：首句缺少关键判别信息

- "因为减免政策要有的" —— 缺少上下文，无法判断是协商还款还是费用咨询
- "电话响一声就挂断了" —— 缺少投诉/停止的明确表态
- "我想咨询一下" —— 过于笼统，需要后续对话

**3. 多意图混合（~20%）**

特征：客户同时表达多种诉求

- "我想提前还款，并要求减免费用" —— 提前还款 + 费用减免
- "我的征信报告显示是关注状态，我想解除关注状态" —— 还款 + 征信修改
- "请帮我查看一下我的会员，为什么我没有续过费用" —— 咨询 + 隐含取消

**4. Gold 标注可争议（~10%）**

特征：标注与模型预测都合理

- "我的账单已经处理完了，为什么还在打电话？" —— collection_complaint 或 stop_collection 都合理
- "今天早上众安保险怎么又从我这边扣款了？" —— bill_deduction_query 或 deduction_issues 都合理
